# Auditoría del copy restaurado de Valyrian.js

**Fecha:** 2026-07-22  
**Fecha de aplicación:** 2026-07-22  
**Estado global:** Aprobado y aplicado. Las 68 decisiones se implementaron en las fuentes autoradas EN/ES.  
**Alcance:** Copy visible y autorado en inglés y español después de restaurar el estado editorial anterior a la delegación cancelada. Incluye Home, Learn, Guides, Recipes, API Reference, navegación, búsqueda, errores y estados de interfaz.  
**Criterio:** Conservar únicamente texto que ayude a ejecutar tareas con Valyrian.js, elegir entre capacidades públicas o entender entradas, resultados, errores, estados y restricciones observables. El consejo general y los internals se proponen para eliminación. Solo se reformula cuando la misma fuente respalda una acción o un comportamiento público útil.

**Este documento conserva la evidencia editorial de decisiones aprobadas y aplicadas al copy del sitio.**

## Inventario de superficies revisadas

- `client/i18n/ui.en.js` y `client/i18n/ui.es.js`.
- `client/app/site.js` y la prosa visible definida en `client/pages` y `client/components`.
- `content/en/guides.md`, `content/es/guides.md`, `content/en/recipes.md` y `content/es/recipes.md`.
- `content/view-content.js`, incluidas rutas de aprendizaje, grupos, guías, recetas y API Reference.
- `server/index.js` y los literales de `build.js` que podían llegar a una superficie visible.

Se excluyeron dependencias, snippets, firmas, tipos, identificadores de API, pruebas, reportes previos y archivos generados. `public/generated/pages.en.json`, `public/generated/pages.es.json` y `public/app.js` se usaron únicamente como evidencia de restauración.

## Criterios para una futura aplicación

1. Aplicar decisiones únicamente en las fuentes autoradas y regenerar después mediante el pipeline existente.
2. Mantener paridad conceptual EN/ES en acción, alcance y certeza.
3. Conservar advertencias solo cuando nombren una entrada, estado, error, precondición o consecuencia observable de una API pública.
4. Eliminar consejos generales de seguridad, UX, arquitectura, organización, infraestructura o proceso.
5. Sustituir ownership, reconciliación, colecciones, registros y temporizadores internos por llamadas, entradas y resultados públicos.
6. No usar el reporte como autorización para cambiar el sitio.
7. No añadir marketing, benchmarks, adopción ni comportamiento no respaldado.
